# Echo Maze v5.0 总体修改计划

## 前置：现状与目标对照

| | v4.1 (现状) | v5.0 (目标) |
|---|-----------|-----------|
| 叙事 | 静默吞噬村子，六位回声行者，存在主义 | 卡德摩斯站声场事故，六名同事，工程阴谋 |
| 关卡 | 27关，Boss各3次 | 10关，Boss各1次 |
| 战斗 | 近身ping3次 | 双重脉冲（探测/聚焦），3层护盾 |
| 角色 | 六种回声行者人格 | 六个人的外骨骼遗物 |
| 碎片 | 记忆碎片=导航道具 | 叙事碎片=收集品18块 |
| UI | 迷宫/神话主题 | 设施/工业/外骨骼主题 |
| 物品/环境 | 瓦斯云/水晶簇/暗水/回音壁 | 保留但规则兼容聚焦脉冲 |

---

## 阶段总览

```
Phase 0: 知识库更新 (1天)
  └─ 更新所有设计文档为权威参考

Phase 1: 后台数据层 (2-3天)
  └─ server.py: 关卡结构/碎片/Boss/角色/色板

Phase 2: 核心机制层 (3-4天)
  └─ echo_maze.py: 双重脉冲/护盾/敌人行为

Phase 3: 前端交互层 (2-3天)
  └─ app.js: 聚焦蓄力/Boss UI/碎片面板

Phase 4: 界面视觉层 (2-3天)
  └─ index.html + style.css + pixel_icons.js

Phase 5: 叙事内容层 (2-3天)
  └─ 18块碎片文本 + 10关开场白 + Boss对话 + 终章

Phase 6: 测试调优 (1-2天)
  └─ 关卡参数/能量平衡/Boss难度
```

**总计估算**：约12-15天（单人全职），实际按可用时间调整

## Phase 2：核心机制层 — echo_maze.py

**目的**：实现双重脉冲系统、Boss护盾规则、普通敌人不可击杀规则、环境元素兼容聚焦脉冲。

### 2.1 双重脉冲

**涉及代码**：`echo_maze.py` 中的 `ping()` 函数或等效脉冲逻辑

| # | 变更 |
|---|------|
| 2.1a | `ping()` 拆分为 `scan_pulse()` 和 `focus_pulse()` |
| 2.1b | `scan_pulse()`：消耗1能量，范围7格（受角色修正），照亮墙壁 |
| 2.1c | `focus_pulse()`：消耗3能量，范围前方3格扇形，造成伤害 |
| 2.1d | 客户端传递按键时长参数：`duration < 0.2` → 扫描，`duration >= 0.5` → 聚焦 |
| 2.1e | 0.2-0.5秒之间的按键：降级为扫描脉冲（防误操作） |
| 2.1f | 能量不足3时，`focus_pulse()` 返回失败状态（客户端显示"能量不足"） |

### 2.2 Boss护盾系统

**涉及代码**：`echo_maze.py` 中的Boss类

| # | 变更 |
|---|------|
| 2.2a | Boss属性从 `hp` 改为 `shields`（护盾层数=3） |
| 2.2b | `take_damage()` 只在伤害来源=聚焦脉冲时生效 |
| 2.2c | 每次护盾碎裂→触发 `on_shield_break()` 回调（掉落水晶+叙事文本） |
| 2.2d | 回响之母：`take_damage()` 加距离检查（>3格无效） |
| 2.2e | 静默猎手：加额外护盾逻辑（每吃1个水晶+1层） |
| 2.2f | 虚空化身：加锚点检测逻辑（只在锚点位置可伤害） |

### 2.3 普通敌人规则

| # | 变更 |
|---|------|
| 2.3a | 所有无人机敌人标记为 `invulnerable_to_focus = True` |
| 2.3b | 聚焦脉冲击中普通敌人→弹飞+晕眩5秒，不击杀 |
| 2.3c | 增加敌人类型：`FAULT`（第8关故障型）、`SLEEPING_GIANT`（第7关巨兽） |
| 2.3d | 第7关步数计数器+第51步触发巨兽醒来 |

### 2.4 环境元素兼容

| # | 变更 |
|---|------|
| 2.4a | 瓦斯云：探测脉冲→扩散，聚焦脉冲→穿透点燃 |
| 2.4b | 水晶簇：探测脉冲→折射，聚焦脉冲→摧毁+全屏闪 |
| 2.4c | 暗水：两种脉冲都无效，水中每5秒恢复2能量 |
| 2.4d | 回音壁：探测脉冲→反弹显示，聚焦脉冲→方向随机化 |

### 2.5 能量经济保底

| # | 变更 |
|---|------|
| 2.5a | 能量归零后，每秒自动恢复0.5能量（仅够探测脉冲） |
| 2.5b | Boss战中，每次护盾碎裂掉落1-2水晶（确保续航） |

## Phase 3：前端交互层 — app.js

**目的**：实现聚焦脉冲的按键识别、Boss护盾显示、碎片收集UI、终章选择。

### 3.1 聚焦脉冲按键识别

**涉及代码**：`app.js` 中的键盘事件处理

| # | 变更 |
|---|------|
| 3.1a | `keydown` 空格：记录按下时间戳 |
| 3.1b | `keyup` 空格：计算按键时长 `duration = release - press` |
| 3.1c | `duration < 0.2`：发送 `action: 'scan'` |
| 3.1d | `duration >= 0.5`：发送 `action: 'focus'` |
| 3.1e | `0.2 <= duration < 0.5`：发送 `action: 'scan'`（降级） |
| 3.1f | 蓄力期间（duration >= 0.2），客户端显示充电环动画 |

### 3.2 充电环动画

| # | 变更 |
|---|------|
| 3.2a | 在 `renderPixelFrame()` 流程中增加充电环叠加层 |

| 3.2c | 环的颜色映射当前能量：蓝(高) / 黄(中) / 红(低) |
| 3.2d | 提前释放或释放聚焦脉冲→环消失 |

### 3.3 Boss护盾UI
